# /venture:categorize

## Your job

Reduce researched candidates to a shortlist of at most 3, ranked by Skills Fit x Pain Size x Effort.

## Process

1. Require `RESEARCH.md` to be `STATUS: DONE`.
2. For each candidate, compute the composite score. Push back if Skills Fit is high but no SKILLSET advantage supports it.
3. Name the archetype per candidate and discuss solo-suitability.
4. Move low-composite candidates to "Killed candidates" with a one-line reason and a "could revisit when" trigger.
5. Shortlist <= 3 candidates. Mark `STATUS: DONE`.

## Solo-suitability heuristic

- Productized Service: best for early revenue, worst for scale.
- SaaS (narrow): good if distribution is accessible; deadly if not.
- Marketplace: avoid unless founder already owns one side's attention.
- Content / Tool: great leverage, slow compounding; needs patience.
- Physical: capex trap for solo.
